# 生成思维导图技能

根据用户提供的内容或需求生成思维导图。

生成思维导图需要使用MCP服务：`simple-mind-map-mcp`。如果已经安装了这个服务，那么你可以根据下面的流程来进行。如果没有安装，那么这个技能无法使用。

## 工作流程

### 第一步：打开思维导图编辑器

调用服务的`start_editor`工具，该操作会在浏览器打开一个新标签页，加载思维导图编辑器页面。

### 第二步：检查和生成思维导图内容

如果用户提供了要生成的思维导图的具体内容，你需要先判断是否是`Markdown`格式。如果是，那么执行第三步。如果否，想要提示用户输入正确的`Markdown`格式的内容。

如果用户没有提供具体内容，但是提供了想要生成的思维导图的主题或话题，那么你需要根据这个主题或话题生成`Markdown`格式的思维导图内容，然后执行第三步。

如果用户既没有提供内容，也没有提供主题，那么你需要追问用户。

### 第三步

根据用户提供或者你生成的`Markdown`格式的内容，调用服务的`set_mind_map_data`工具来生成思维导图。

`set_mind_map_data`工具存在三个参数：

- `data`：必传，`Markdown`格式的思维导图内容。

- `layout`：可选，如果用户指定了要使用什么结构，那么需要根据用户的指定结构在可选结构中选择一个最符合用户需求的结构。否则，该参数不需要传递。

- `theme`：可选，如果用户指定了要使用什么样式主题，那么需要根据用户的指定主题在可选主题中选择一个最符合用户需求的主题。否则，该参数不需要传递。

设置完思维导图数据后，接下来可以仅更新思维导图的样式主题或结构。

仅更新样式主题可以调用服务的`set_mind_map_theme`工具。

仅更新结构布局可以调用服务的`set_mind_map_layout`工具。

能通过`set_mind_map_data`工具一次完成的操作不需要拆分成多个工具调用。

最佳实践：

`set_mind_map_data('# 思维导图', 'logicalStructure', 'oreo')`

反面示例：

```
set_mind_map_data('# 思维导图', 'logicalStructure')
set_mind_map_theme('oreo')
```

注意：

1.如果用户没有明确让你读取当前项目的文件，不要进行该操作。

2.从始至终无需在本地生成任何文件。

## 所有可用工具

- `start_editor` - 启动思维导图编辑器
- `set_mind_map_data` - 全量更新编辑器渲染的思维导图数据
- `set_mind_map_theme` - 修改编辑器渲染的思维导图样式主题
- `set_mind_map_layout` - 修改编辑器渲染的思维导图结构布局
- `close_editor` - 关闭当前打开的思维导图编辑页面
- `export_mind_map_to` - 将当前渲染的思维导图导出为指定类型的文件
